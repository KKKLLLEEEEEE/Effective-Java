# Effective Java 学习仓库编码规范

## 目的
本仓库用于学习和演示 Effective Java 原则的代码示例。代码应清晰、具有教育意义并遵循最佳实践。

## 代码组织

### 包结构
- 按 Effective Java 章节组织示例
- 使用包命名：`chapter[编号]`（例如：`chapter2`、`chapter3`）
- 每个章节下按条目创建子目录：`chapter[编号]/item[编号]`
- 例如：`chapter2/item1/`、`chapter2/item2/`

### 文件命名
- 使用 `Item[编号]_[主题].java` 格式（例如：`Item1_StaticFactory.java`）
- 对比示例在同一文件中用内部类区分Good/Bad
- 文件名应简洁且反映条目核心内容

## Java 代码规范

### 文档
- 每个类必须有 Javadoc 注释说明：
  - 演示的是哪个 Effective Java 条目
  - 所阐述的关键原则
  - 任何重要的注意事项
- 包含 `@author` 标签
- 添加行内注释突出显示关键原则

### 代码风格
- 遵循标准 Java 命名约定
- 保持示例简洁，专注于特定原则
- 优先考虑可读性
- 使用有意义的变量和方法名

### 示例要求
- **必须使用动物相关的例子**（如狗、猫、鸟等），保持整个仓库的一致性
- 应用所演示的 Effective Java 原则
- 在有助于学习时展示正确和错误的方法
- 用具体示例演示原则的重要性

### 应避免的事项
- 不要在一个示例中混合多个不相关的条目
- 避免过于复杂而掩盖原则的示例
- 除非是条目的重点，否则不要包含生产级错误处理
- 最小化依赖 - 保持示例自包含

## 示例模板

```java
package chapter[编号].item[编号];

/**
 * 演示 Effective Java 第[编号]条：[标题]
 * 
 * 核心原则：[简要说明]
 * 
 * @author CN-JeffreyZhou
 */
public class Item[编号]_[主题] {
    // 清晰、专注的代码演示原则
}
```

## 测试
- 包含简单的 main() 方法演示原则的实际应用
- 使用打印语句验证结果和说明要点

## 注释和学习笔记
- 添加注释解释为什么使用某个模式
- 在注释中引用具体的 Effective Java 条目编号
- 包含关于常见陷阱或误解的简要说明
